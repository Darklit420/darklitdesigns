    Our Story

    Why we built
    Darklit.

    The internet has a brightness problem. We built the fix.

    The Problem

    The internet is
    too bright.

    Open any website. Any of them. Nine times out of ten you're looking at a white background with dark text. A design choice made in the early days of the web because monitors were dim and white backgrounds helped with legibility. That was 1994. Monitors are not dim anymore. The problem those white backgrounds were solving stopped existing thirty years ago.

    But the white backgrounds stayed. They became the default. Then the default became the expectation. Then the expectation became so deeply embedded in web design culture that building something that looked different felt like breaking a rule. Dark mode was a toggle you added at the end if you had time. Light mode was the real site. Dark mode was the afterthought.

    We disagreed with that. We disagree with it completely.

    Dark mode isn't an accessibility feature or a preference toggle. It's a fundamentally better way to present visual information on a screen that emits light. The contrast works differently. The colors pop differently. The whole experience of looking at a site for more than thirty seconds is different — and better — when you're not staring into a light source set to full brightness.

    We wanted to build something that treated dark mode as the starting point. Not the afterthought. Not the toggle. The foundation.

    The Solution

    One file.
    Both worlds.

    The idea behind Darklit started with a simple question — what if dark mode and light mode weren't two different things you had to build separately? What if they were two sides of the same design system, built at the same time, with the same care, existing in the same file?

    That question became the architecture. Every color palette in Darklit has two complete versions — a dark world and a light world. Not dark mode as an inverted version of light mode. Not light mode as a washed-out version of dark mode. Two fully realized, independently designed color environments that share the same structure but feel completely different from each other. The dark version of Pacific feels like 11pm at Torrey Pines. The light version feels like the same beach at noon. Same place. Different light. Different feeling. Both intentional.

    Then we added the editor. Because the best color system in the world is worthless if you can't make it yours. Add ?edit=true to any Darklit URL and the full style editor opens — colors, fonts, backgrounds, hero gradients, dark and light mode independently. Make your changes, hit Export, and you get back one clean HTML file that has everything baked in. No build tools. No deployment pipeline. No server required. Open it in a browser and your site is live.

    That's the one file philosophy. Everything you need to run a professional website lives in a single HTML file that you own outright. No platform. No subscription. No monthly fee. No moment where someone else's business decision affects whether your site stays online. You have the file. The file is yours. The site is yours forever.

    The City Editions

    Colors from
    real places.

    At some point during development we started asking a different question. Not just "what colors look good" but "what colors mean something." Generic color palettes are everywhere. Every design tool in the world ships with twenty of them and they're all fine and none of them feel like anything in particular.

    We wanted palettes that felt like somewhere. Specifically somewhere. The Pacific palette isn't "deep blue with cyan accents" — it's bioluminescent waves at Torrey Pines at 11pm, where the water glows electric against near-black and the horizon disappears entirely into the Pacific. The Gaslamp palette isn't "neon pink and purple" — it's Saturday night on 5th Avenue in 1887 Victorian brick buildings that are now lit by signs that weren't there when they were built. Every color choice has a reason. Every reason is a real place.

    That became the City Editions concept. Location-based color systems built from real neighborhoods. San Diego was first — seven palettes from seven neighborhoods, each one researched, each one true to what the place actually looks, feels, and lives like. Miami was second — five palettes from five neighborhoods that couldn't be more different from each other or from San Diego. More cities are coming.

    Nothing like this exists anywhere else in web design. We know because we looked. The closest thing is "inspired by" palettes in design tools that are inspired by a vague aesthetic category. We're not inspired by a category. We're built from a specific Tuesday afternoon on a specific street in a specific city. That's a different thing.

    Where We're Going

    This is just
    the beginning.

    The current version of Darklit is a single HTML file with a complete style editor, dark and light mode, city-based color palettes, and a template library of real sites built on the same engine. That's what exists today. What exists today is already more than anything else at this price point — but it's not what we're building toward.

    The photo editor is next. The biggest gap in the current product is image management — right now you have to handle your own photos outside the file. The photo editor will change that. Upload photos directly to your Darklit site, place them anywhere, manage them in-page without touching a line of code. When it launches, the Full Bundle gets it first.

    After that, more cities. New York is next — Brooklyn, Harlem, SoHo, the Financial District, the Village. Atlanta after that. New Orleans. Chicago. Every city on earth has a visual identity waiting to be translated into a design system. We intend to translate all of them, one at a time, with the same care we brought to San Diego and Miami.

    And the template library keeps growing. Every real site we build for a client becomes a template that anyone can buy. HoopHQ was a sports training facility in San Diego. M&E was a talent booking platform. Every industry we touch adds to the library. The goal is a template for every type of small business that exists — and a palette from the city where that business lives.

    The plain white internet is going away. Maybe not today. Maybe not this year. But the direction is clear and we intend to be at the front of it.
